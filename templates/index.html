User Manual

Welcome to the User Manual for the Dynamic IoT Dashboard for Real-time Device Monitoring and Control. This manual will guide you through the usage of the dashboard, from loading data to calculating the energy usage and billing.

1. Dashboard Overview

The dashboard allows you to monitor and control up to 100 Tuya WiFi Smart Sockets in real-time. Each room has its own card on the dashboard displaying:

  • Total Unit (kWh)
  • Total Cost (TK)
  • Switch to toggle the device ON/OFF

Clicking the "Show Full Data" button will display detailed information like voltage, current, and power consumption over a specified time range.

2. Data Loading

The data on the dashboard is dynamically loaded from the Tuya WiFi Smart Sockets every 8 seconds. This is done by sending requests to the Tuya cloud API and fetching data for voltage, current, and power. The data is stored in a MySQL database for each device in different rooms.

When the dashboard is loaded, data for all rooms is fetched and displayed on the dashboard in real-time. The data is also continuously updated in the background.

3. Energy Consumption Calculation

The energy consumption for each room is calculated in "kilowatt-hours (kWh)". Here's how it's done step by step:

  1. Each data sample includes the current wattage consumption for the device in the room.
  2. The system calculates the time difference between consecutive data points.
  3. Using the trapezoidal rule, the average wattage between two data points is used to calculate the energy consumption for that period in kWh.
  4. The total energy consumption is the sum of all periods (based on the number of data points) for that room.

For example:
- If the device is consuming 100W for 1 hour, it will consume 0.1 kWh (100W * 1hr / 1000).
- The system sums these values for all intervals to get the total energy usage.

4. Energy Consumption Example

Let's walk through a step-by-step "Energy Consumption" example:

  1. Total Duration: The total duration of your data is calculated by multiplying the total number of readings by the time interval.
    Formula: Total Duration (Hours) = (Total Readings × Time Interval) / 3600 seconds per hour
    Example: (6290 readings × 8 seconds) / 3600 ≈ 13.98 hours
  2. Average Power: Average power is calculated by summing all watt readings and dividing by the total number of readings.
    Formula: Average Power (Watts) = Sum of all Watt Readings / Total Readings
    Example: 523719.1 / 6290 ≈ 83.26 watts
  3. Total Energy (kWh): The total energy consumed is the product of average power and total duration.
    Formula: Total kWh = (Average Power (W) × Total Duration (H)) / 1000
    Example: (83.26 × 13.98) / 1000 ≈ 1.16 kWh

5. Cost Calculation

The cost for each room is calculated based on the energy consumption and the unit cost in "Bangladeshi Taka (TK)". Here's how the cost is calculated step by step:

  1. Once the total energy consumption (in kWh) is calculated, the system applies a tiered pricing model based on the total units consumed.
  2. The price for each unit is different depending on the amount of energy consumed. Below is the pricing range:
Unit Range (kWh) Price per Unit (TK)
0 - 75 5.26
76 - 200 7.20
201 - 300 7.59
301 - 400 8.02
401 - 600 12.67
Above 600 14.61

For example:
- If the total energy consumption is 80 kWh, the cost calculation would be:
- First 75 kWh is charged at 5.26 TK/unit = 75 * 5.26 = 394.5 TK.
- The remaining 5 kWh is charged at 7.20 TK/unit = 5 * 7.20 = 36.0 TK.
- Total cost = 394.5 + 36.0 = 430.5 TK.

6. Data Filtering

You can filter the data by selecting a custom date range using the "From Date" and "To Date" fields. Once a date range is selected, the system will recalculate the energy consumption and cost for that range and display it in the modal. You can also download the filtered data in CSV format by clicking the "Download CSV" button.

7. Download CSV

The system allows downloading data for each room in CSV format. If the file size exceeds 2MB, the system automatically splits the data into multiple files for easier download.

8. Troubleshooting

In case of any issues with the Tuya API or data fetching, the following steps can be taken:

  • Ensure that the API keys are valid and have not expired.
  • Check if the Tuya WiFi Smart Sockets are connected to the cloud and are functioning properly.
  • Make sure the MySQL database is accessible and not overloaded with queries.

9. Conclusion

This system allows for efficient monitoring and control of energy consumption across 100 devices, providing real-time data and billing information. It offers insights into energy usage and helps reduce costs through better management of electricity consumption.
